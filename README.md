## 🎯 项目概述

### 什么是目录扫描管理系统？

目录扫描管理系统是一个专业的文件系统管理工具，它能够帮助用户：

- **自动扫描**：快速扫描电脑中的文件夹和文件
- **智能管理**：对扫描结果进行分类、标记和管理
- **多用户支持**：支持多个用户独立使用，数据互不干扰
- **图片预览**：自动识别并预览目录中的图片内容
- **智能搜索**：支持中文拼音搜索，快速定位目标
- **数据备份**：自动备份重要数据，防止意外丢失

### 🎨 设计理念

本软件采用 **"简单易用，功能强大"** 的设计理念：

- **面向新手**：界面直观，操作简单，无需专业知识
- **面向专业用户**：提供高级设置和自定义选项
- **面向企业用户**：支持多用户、数据安全、批量操作

### 💡 核心价值

1. **时间节省**：自动扫描代替手动查找
2. **效率提升**：智能搜索快速定位文件
3. **数据安全**：多重备份保障数据完整性
4. **团队协作**：多用户支持便于团队使用
5. **长期维护**：持续更新，功能不断完善

## 📚 基础概念解析

### 🔍 核心概念说明

**1. 主目录 (Main Directory)**
- **通俗理解**：你要管理的"大本营"文件夹
- **专业解释**：作为扫描起点的根目录，系统会从这个目录开始扫描所有子文件夹和文件
- **举例说明**：
  - 个人用户：设置"我的文档"或"图片"文件夹为主目录
  - 办公用户：设置"项目资料"文件夹为主目录
  - 摄影用户：设置"摄影作品"文件夹为主目录

**2. 扫描深度 (Scan Depth)**
- **通俗理解**：要扫描到第几层文件夹
- **专业解释**：从主目录开始，扫描的子目录层级数
- **数值说明**：
  - 深度 0：只扫描主目录本身
  - 深度 1：扫描主目录及其直接子文件夹
  - 深度 3：扫描主目录及其下3层子文件夹
- **举例说明**：
  ```
  主目录: D:/工作资料
  ├── 项目A (深度1)
  │   ├── 设计稿 (深度2)
  │   └── 文档 (深度2)
  ├── 项目B (深度1)
  └── 会议记录 (深度1)
  
  设置扫描深度为2，将扫描到"设计稿"和"文档"文件夹
  ```

**3. 扫描模式 (Scan Mode)**
- **仅扫描目录**：只记录文件夹，忽略文件
- **仅扫描文件**：只记录文件，忽略文件夹
- **扫描目录和文件**：同时记录文件夹和文件

**4. 拼音搜索 (Pinyin Search)**
- **功能说明**：支持用拼音首字母搜索中文名称
- **使用示例**：
  - 搜索"wj"可以找到"文件夹"
  - 搜索"xxt"可以找到"学习资料"
  - 搜索"gssx"可以找到"高等数学"

**用户数据库**

- 便于视觉化浏览

## 🚀 完整使用流程

### 第一阶段：初始设置

#### 步骤1：首次启动与用户创建

**详细操作**：

1. **启动软件**
   - 双击桌面图标或可执行文件
   - 等待软件初始化完成

2. **创建用户账户**
   - 点击"用户管理"按钮
   - 选择"添加用户"
   - 输入用户名（建议使用英文或拼音）

   **用户名建议**：
   - 个人使用：姓名拼音，如"zhangsan"
   - 工作使用：部门+姓名，如"design_zhang"
   - 家庭使用：家庭成员，如"family"

3. **登录用户**
   - 选择刚创建的用户
   - 点击"登录"按钮
   - 系统会为该用户创建独立数据库

#### 步骤2：主目录设置

**添加主目录**：

1. **打开主目录管理**
   - 点击工具栏的"主目录管理"按钮
   - 或通过菜单访问该功能

2. **添加目录**
   - 点击"浏览"选择文件夹
   - 设置显示名称（便于识别）
   - 设置扫描深度（建议3-5）

   **深度设置建议**：
   - 文档管理：深度2-3
   - 图片整理：深度3-5  
   - 项目文件：深度4-6
   - 整个磁盘：深度不建议超过5

3. **保存设置**
   - 点击"添加"按钮
   - 系统会保存主目录配置

### 第二阶段：基础扫描

#### 步骤3：执行首次扫描

**开始扫描**：

1. **选择扫描方式**
   - 单个目录扫描：针对特定主目录
   - 批量扫描：扫描所有已添加的主目录

2. **监控扫描进度**
   - 观察进度条显示
   - 查看状态栏信息
   - 等待扫描完成提示

3. **查看扫描结果**
   - 扫描完成后自动显示结果
   - 检查目录数量和状态
   - 验证数据完整性

#### 步骤4：浏览和管理结果

**结果浏览**：

1. **表格视图**
   - 封面列：显示目录的代表图片
   - 目录名称：文件夹或文件名称
   - 状态：上架(存在)/下架(不存在)
   - 路径：完整文件路径

2. **图片预览**
   - 点击封面图片查看大图
   - 浏览其他缩略图
   - 支持多图片切换查看

3. **状态管理**
   - 绿色"上架"：文件存在且可访问
   - 红色"下架"：文件已被删除或移动

### 第三阶段：日常使用

#### 步骤5：搜索与查找

**搜索技巧**：

1. **基础搜索**
   - 在搜索框输入关键词
   - 支持中文、英文、数字
   - 实时显示匹配结果

2. **拼音搜索**
   - 输入拼音首字母
   - 如"wj"搜索"文件夹"
   - 如"txgl"搜索"图像管理"

3. **路径搜索**
   - 搜索完整或部分路径
   - 如搜索"D:/工作"找到相关工作文件夹

#### 步骤6：数据维护

**定期维护**：

1. **重新扫描**
   - 定期点击"扫描目录"更新数据
   - 或依赖自动扫描功能

2. **数据备份**
   - 定期手动备份重要数据
   - 检查自动备份是否正常

3. **清理数据**
   - 清除不存在的目录记录
   - 优化数据库性能

### ⚙️ 设置功能详解

#### 扫描设置

**扫描间隔**
- **作用**：控制自动扫描的频率
- **建议值**：
  - 频繁变动：30-60分钟
  - 一般使用：2-4小时
  - 稳定环境：8-24小时

**自动扫描**
- **启用**：按间隔自动扫描
- **禁用**：完全手动扫描
- **适用场景**：
  - 启用：文件经常变动
  - 禁用：文件稳定，手动控制

**扫描模式**
- **仅扫描目录**：适合管理文件夹结构
- **仅扫描文件**：适合管理具体文件
- **扫描目录和文件**：完整管理方案

**目录过滤**
- **包含过滤**：只扫描包含关键词的目录
  - 示例："project,work" 只扫描含project或work的文件夹
- **排除过滤**：跳过包含关键词的目录
  - 示例："temp,backup" 跳过临时和备份文件夹

#### 封面相关设置

**扫描时保存封面**
- **开启**：每次扫描自动更新封面
- **关闭**：手动控制封面保存
- **资源消耗**：开启会增加扫描时间

**封面备份存在则跳过**
- **开启**：封面已存在时跳过保存
- **关闭**：总是重新保存封面
- **建议**：开启以提升效率

**封面保存模式**
- **智能模式**：自动识别AV格式使用系列模式
- **系列模式**：按XXXX-####格式建立系列目录
- **首字模式**：按名称首字母建立目录

**封面保存目录**
- **自定义目录**：指定封面保存位置
- **默认目录**：用户主目录下的"目录封面"文件夹

### 👥 用户管理详解

#### 用户操作

**添加用户**
- 创建新的独立账户
- 每个用户数据完全隔离
- 支持个性化设置

**重命名用户**
- 修改用户显示名称
- 不影响用户数据
- 立即生效

**删除用户**
- **警告**：同时删除所有用户数据
- **不可恢复**：操作前请备份
- **确认机制**：双重确认防止误操作

**清除用户数据**

## ❓ 常见问题

### 🤔 基础使用问题

**Q: 软件是免费的吗？**
A: 是的，本软件基于MIT许可证完全免费开源使用。

**Q: 支持哪些操作系统？**
A: 支持Windows 7+、macOS 10.12+、Linux等主流操作系统。

**Q: 需要安装Python吗？**
A: 如果使用可执行文件版本不需要，使用源代码版本需要Python 3.6+。

**Q: 软件支持中文吗？**
A: 完全支持中文界面、中文搜索、中文路径处理。

### 🔧 功能相关问题

**Q: 最多支持多少个主目录？**
A: 理论上无限制，但建议根据实际需要添加，一般10-20个为宜。

**Q: 扫描深度设置多少合适？**
A: 文档管理2-3层，图片管理3-5层，深层结构不超过6层。

**Q: 自动扫描会影响电脑性能吗？**
A: 扫描时会占用一定资源，建议设置在电脑空闲时进行。

**Q: 支持网络驱动器扫描吗？**
A: 支持，但扫描速度可能较慢，建议增加扫描间隔。

### 💾 数据管理问题

**Q: 数据存储在哪里？**
A: 用户数据存储在用户目录下的应用数据文件夹中。

**Q: 如何迁移数据到新电脑？**
A: 备份数据库文件，在新电脑上恢复即可。

**Q: 支持多大体积的文件库？**
A: 理论上无限制，但建议单个数据库不超过10GB。

**Q: 误删除了用户如何恢复？**
A: 如果备份文件存在，可以通过备份恢复用户数据。

### 🖼️ 图片相关问题

**Q: 支持哪些图片格式？**
A: 支持PNG、JPG、JPEG、GIF、BMP等常见格式。

**Q: 封面图片如何选择？**
A: 优先使用cover.*文件，其次按文件名排序的第一张图片。

**Q: 封面保存会占用多少空间？**
A: 通常每个封面50-200KB，1万个封面约0.5-2GB。

**Q: 可以自定义封面吗？**
A: 当前版本自动选择，手动替换封面文件也可生效。

### 🔍 搜索相关问题

**Q: 拼音搜索准确吗？**
A: 准确率95%以上，支持多音字和常见词汇。

**Q: 支持正则表达式搜索吗？**
A: 当前版本不支持正则表达式，支持通配符和拼音搜索。

**Q: 搜索速度如何？**
A: 万级记录搜索响应时间在1秒以内。

**Q: 可以搜索文件内容吗？**
A: 当前版本只搜索文件名和路径，不搜索文件内容。
