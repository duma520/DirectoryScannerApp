## 🎯 项目概述

### 1.1 什么是目录扫描管理系统？

目录扫描管理系统是一个专业的文件系统管理工具，它能够自动扫描、组织和展示您计算机中的目录和文件结构。就像给您的电脑文件系统装上了一双"智能眼睛"，让您能够：

- **快速找到**任何文件夹或文件
- **可视化浏览**目录内容
- **智能搜索**中文和拼音
- **批量管理**多个主目录
- **自动备份**重要数据

### 1.2 设计理念

本软件的设计遵循"简单易用，功能强大"的原则：

- **对新手友好**: 直观的界面，一键式操作
- **对专业人士强大**: 支持自定义扫描深度、过滤条件等高级功能
- **跨平台兼容**: 支持 Windows、macOS、Linux
- **数据安全**: 自动备份，防止数据丢失

### 1.3 核心价值

**对于个人用户**：
- 整理杂乱的文件目录
- 快速找到忘记位置的文件
- 管理照片、文档集合

**对于办公人员**：
- 管理项目文件夹
- 团队文件目录共享
- 定期扫描监控重要目录

**对于IT专业人员**：
- 系统目录监控
- 批量文件管理
- 自动化目录维护

## 📚 基础概念解释

### 5.1 什么是"目录"和"文件"？

**简单理解**：
- **目录（文件夹）**：就像现实中的文件夹，用来装东西的容器
- **文件**：就像文件夹里装的具体文件，比如照片、文档

**举例说明**：
```
我的电脑
├── C盘 (目录)
│   ├── 用户 (目录)
│   │   ├── 张三 (目录)
│   │   │   ├── 照片 (目录)
│   │   │   │   ├── 假期.jpg (文件)
│   │   │   │   └── 生日.png (文件)
│   │   │   └── 文档 (目录)
│   │   │       └── 报告.docx (文件)
```

### 5.2 什么是"扫描深度"？

**简单解释**：
扫描深度决定程序会查看多少层文件夹。

**举例说明**：
假设目录结构：
```
项目/
├── 文档/
│   ├── 设计/
│   │   └── 初稿.txt
│   └── 需求/
│       └── 说明.docx
└── 代码/
    └── 源码/
        └── main.py
```

- **深度1**：只看到 `项目/文档` 和 `项目/代码`
- **深度2**：还能看到 `项目/文档/设计` 和 `项目/文档/需求`
- **深度3**：能看到所有文件和文件夹

### 5.3 什么是"主目录"？

主目录是您想要重点监控和管理的核心文件夹。比如：
- 您的"文档"文件夹
- 公司的"共享项目"文件夹
- 您的"照片库"文件夹

### 5.4 数据库是什么？

**简单理解**：
数据库就像程序的"记忆本"，记录着：
- 扫描过哪些文件夹
- 文件夹里有什么
- 什么时候扫描的
- 用户设置信息

**重要提示**：数据库文件通常保存在程序目录下，请不要随意删除。

## 🚀 功能详解

### 6.1 用户管理功能

#### 6.1.1 创建用户

**为什么要多用户？**
- 家庭成员共享电脑但数据分开
- 工作个人数据分离
- 不同项目使用不同账户

**操作步骤**：
1. 启动程序
2. 点击"用户管理"
3. 输入用户名（如"张三的工作文件"）
4. 点击"添加用户"
5. 选择用户并登录

**专业提示**：
- 用户名最好具有描述性
- 定期备份用户数据
- 重要数据使用独立用户

#### 6.1.2 用户数据管理

**清除用户数据**：
- 只删除扫描记录，不删除实际文件
- 用户账户仍然保留
- 适用于重新开始扫描

**删除用户**：
- 完全删除用户所有数据
- 包括扫描记录和设置
- **不可恢复**，请谨慎操作

### 6.2 主目录管理

#### 6.2.1 添加主目录

**操作步骤**：
1. 点击"主目录管理"
2. 点击"浏览"选择文件夹
3. 输入显示名称（可选）
4. 设置扫描深度（推荐3-5）
5. 点击"添加"

**深度设置建议**：
- **深度1-2**：快速扫描，适合监控顶层变化
- **深度3-5**：平衡深度和性能，适合大多数场景
- **深度6+**：深度扫描，可能较慢但完整

#### 6.2.2 管理多个主目录

**典型配置**：
```
主目录列表：
1. [工作项目] 路径: D:\Projects, 深度: 4
2. [个人照片] 路径: E:\Photos, 深度: 3  
3. [学习资料] 路径: C:\Study, 深度: 2
```

**批量操作**：
- "扫描所有主目录"：一次性扫描所有配置的目录
- 单个扫描：只扫描选定的主目录

### 6.3 扫描功能

#### 6.3.1 扫描模式详解

**三种扫描模式**：

1. **仅扫描目录**（推荐给新手）
   - 只记录文件夹信息
   - 忽略所有文件
   - 扫描速度快
   - 适合管理文件夹结构

2. **仅扫描文件**
   - 只记录文件信息
   - 忽略空文件夹
   - 适合文件搜索场景

3. **扫描目录和文件**（功能最全）
   - 记录所有内容
   - 扫描速度较慢
   - 数据最完整

#### 6.3.2 自动扫描

**设置方法**：
1. 点击"设置"
2. 在"扫描设置"中启用自动扫描
3. 设置扫描间隔（分钟）
4. 保存设置

**间隔建议**：
- **15-30分钟**：高频率监控，适合重要项目
- **1-2小时**：平衡模式，推荐使用
- **4小时以上**：低频率，适合不常变化的目录

### 6.4 搜索功能

#### 6.4.1 基础搜索

**支持搜索的内容**：
- 目录名称
- 文件名称
- 完整路径

**搜索技巧**：
- 不区分大小写："DOCUMENT" 和 "document" 都能找到
- 部分匹配："项目" 可以找到 "我的项目"、"项目文档"
- 路径搜索：输入路径关键词快速定位

#### 6.4.2 拼音搜索（特色功能）

**什么是拼音搜索？**
即使只记得中文名称的拼音首字母，也能找到目标。

**使用示例**：
- 搜索"wj" → 找到"文件夹"
- 搜索"xm" → 找到"项目"、"姓名"
- 搜索"tp" → 找到"图片"、"投票"

**技术原理**：
程序自动将中文转换为拼音首字母，建立搜索索引。

#### 6.4.3 高级搜索技巧

**组合搜索**：
- 多个关键词用空格分隔
- "项目 2024" 找到包含这两个词的目录

**排除搜索**：
- 使用减号排除特定词
- "项目 -临时" 找到包含"项目"但不包含"临时"的目录

### 6.5 图片预览功能

#### 6.5.1 封面识别

**智能封面选择**：
1. 优先查找 `cover.*` 文件
2. 查找 `封面.*` 文件  
3. 使用目录中第一张图片

**支持的图片格式**：
- JPEG (.jpg, .jpeg)
- PNG (.png)
- GIF (.gif)
- BMP (.bmp)

#### 6.5.2 缩略图展示

**显示逻辑**：
- 主封面：左侧大图显示
- 其他图片：右侧小缩略图显示（最多3张）
- 自动排序：按文件名顺序显示

### 6.6 数据备份与恢复

#### 6.6.1 自动备份

**备份策略**：
- 每次扫描完成后自动备份
- 保留最近30个备份
- 按时间戳命名备份文件

**备份文件位置**：
```
程序目录/
└── backups/
    ├── auto_20241103_143022.db
    ├── manual_20241102_093015.db
    └── restore_pre_20241101_162345.db
```

#### 6.6.2 手动备份

**何时需要手动备份**：
- 重要操作前（如批量删除）
- 程序升级前
- 系统重装前

**操作步骤**：
1. 点击"数据库备份"
2. 选择备份位置（可选）
3. 确认备份

#### 6.6.3 数据恢复

**恢复场景**：
- 误操作导致数据丢失
- 程序异常数据损坏
- 系统崩溃后恢复

**恢复步骤**：
1. 点击"恢复数据库"
2. 从备份列表选择时间点
3. 确认恢复（建议先备份当前状态）

## 🔧 高级功能

### 8.1 过滤设置

#### 8.1.1 包含过滤

**用途**：只关注特定类型的目录

**设置格式**：逗号分隔的关键词

**示例**：
```
包含过滤：project,work,重要
效果：只扫描包含这些关键词的目录
```

#### 8.1.2 排除过滤

**用途**：忽略不必要的目录

**典型排除项**：
- 系统目录：`temp, cache, log`
- 备份目录：`backup, old, 废弃`
- 临时文件：`tmp, 临时`

### 8.2 性能优化

#### 8.2.1 扫描性能调优

**影响因素**：
1. 目录数量和质量
2. 扫描深度设置
3. 扫描模式选择
4. 硬件性能

**优化建议**：
- 重要目录深度设深，次要目录设浅
- 使用"仅扫描目录"模式提高速度
- 避免扫描系统目录和临时文件

#### 8.2.2 内存使用优化

**监控指标**：
- 程序启动内存：~50-100MB
- 扫描时内存：根据数据量增加
- 长期运行：稳定在100-200MB

#### 9.3.1 目录扫描算法

**广度优先搜索(BFS)**：
```python
def scan_directory_with_depth(root_dir, max_depth):
    queue = deque([(root_dir, 0)])  # (路径, 当前深度)
    while queue:
        current_path, current_depth = queue.popleft()
        if current_depth > max_depth:
            continue
        # 处理当前目录
        process_directory(current_path)
        # 添加子目录到队列
        for entry in os.listdir(current_path):
            queue.append((os.path.join(current_path, entry), current_depth + 1))
```
